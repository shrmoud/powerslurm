Week 2 - Oct 10

As the first week came to a close, I made sure that I understood the three cluster/scheduler concepts. there is already the master list which contains all of the jobs in the wait queue. The window is just a selection of the top N jobs from that queue. When we do scheduling, we do it on the window list, not the queue on the whole. We are concerned about the algorithm,0-1 Knapsack, but we only apply that algorithm to the window as it would most surely cause jobs to starve if it worked on the entire queue. I also tried out the Microsoft Azure SLURM Example.

Week 3 - Oct 17

SLURM implements default_queue_depth=# which specifies the number of jobs to consider for scheduling on each event that may result in a job being scheduled. Default value is 100 jobs. Since this happens frequently, a relatively small number is generally best. Thi could be used as our window in our window based scheduling approach. I also found the Google Groups for the different SLURM groups. PySLURM Group and SLURM Developer Group I also tried installing the SLURM Simulator
